1.1 获取头部信息修改

请求方法：POST
需要登录：否
请求地址：http://dev-api.artzhe.com/V31/ArtCircle/getHeader
需要token：是(GET方式传token)

说明

增加返回字段：isShowGallery，1-可以进入艺术家画廊，0-不可以进入艺术家画廊

请求参数列表

字段 字段描述 规格 必须 示例 备注
uid 用户id int 否 - 如果有，则是个人艺术圈动态列表头部信息,返回uid的用户信息；没有，则是艺术圈列表头部信息。另，如果没有登录，艺术圈列表头部信息只有banner

响应结果示例

  1. {
  2.   "data": {
  3.     "status": 1000, //正确时只有这一种情况
  4.     "info": {
  5.       "uid": "100", //用户uid
  6.       "name": "张三", //名字
  7.       "gender": "1", //用户性别1男2女3未知
  8.       "faceUrl": "http://artzhe.oss-cn-shenzhen.aliyuncs.com/test1.png", //头像
  9.       "isArtist": 1, //1表示是艺术家,0表示非艺术家
  10.       "isAgency": 1, //1表示是认证机构,0表示非认证机构
  11.       "agencyType":7,//机构类型
  12.       "isPlanner": 0, //1表示是策划者,0表示非策划者
  13.       "banner": "http://gsy-other.oss-cn-beijing.aliyuncs.com/uploads/2017/05/31/1496199190575.png", //banner
  14.       "unreadMessageTotal": 1, //未读消息总数
  15.       "isShowGallery": 1, //1-可以进入艺术家画廊，0-不可以进入艺术家画廊
  16.     }
  17.   },
  18.   "code": 30000, //其它非30000表示错误的情况
  19.   "message": "success",
  20.   "debug": false
  21. }

说明

增加请求参数，返回数据不变：device,system,version

请求参数列表

字段 字段描述 规格 必须 示例 备注
mobile 手机号码 正则：^1[34578]{1}\d{9}$ 是 "18588260188" -

password 密码 - 否 - -

system 系统信息 string 是 -
version APP版本 string) 是 -

  2.   "data": {
  3.     "status": 1000//跟code参数一起用来判断是否增加新的微信成功
  4.     "inviteCode":"100001",登录成功后的邀请码
  5.     "isArtist":1//1、艺术家，-1，非艺术家

需要token：是(GET方式传token)

说明

增加请求参数，返回数据不变：device,system,version

字段 字段描述 规格 必须 示例 备注
thirdInfo 第三方的登录信息 string(json object) 是 参考thirdInfo示例 -
thirdFullInfo 第三方登录的完整信息 string(json object) 否 "{}" -
device 设备信息 string 是 -
system 系统信息 string 是 -
version APP版本 string) 是 -

thirdInfo示例

  1. {
  2.   "partnerCode":"合作方编码。WECHAT：微信；QQ：QQ；WEIBO：新浪微博",
  3.   "openId":"合作方登录账号开放ID",
  4.   "unionId":"合作方登录账号联合ID",
  5.   "nickname":"昵称，为空值时，从合作方用户信息里取相应的nickname",
  6.   "gender":"性别。1：男；2：女；3：保密",
  7.   "faceUrl":"头像Url"
  8. }

响应结果示例

  1. {
