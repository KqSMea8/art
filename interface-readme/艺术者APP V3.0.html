1 艺术圈

1.1 获取头部信息

请求方法：POST
需要登录：否
请求地址：http://api.artzhe.com/V30/ArtCircle/getHeader
需要token：是(GET方式传token)

请求参数列表

字段 字段描述 规格 必须 示例 备注
uid 用户id int 否 - 如果有，则是个人艺术圈动态列表头部信息,返回uid的用户信息；没有，则是艺术圈列表头部信息。另，如果没有登录，艺术圈列表头部信息只有banner

响应结果示例

  1. {
  2.   "data": {
  3.     "status": 1000, //正确时只有这一种情况
  4.     "info": {
  5.       "uid": "100", //用户uid
  6.       "name": "张三", //名字
  7.       "gender": "1", //用户性别1男2女3未知
  8.       "faceUrl": "http://artzhe.oss-cn-shenzhen.aliyuncs.com/test1.png", //头像
  9.       "isArtist": 1, //1表示是艺术家,0表示非艺术家
  10.       "isAgency": 1, //1表示是认证机构,0表示非认证机构
  11.       "agencyType":7,//机构类型
  12.       "isPlanner": 0, //1表示是策划者,0表示非策划者
  13.       "banner": "http://gsy-other.oss-cn-beijing.aliyuncs.com/uploads/2017/05/31/1496199190575.png", //banner
  14.       "unreadMessageTotal": 1, //未读消息总数
  15.     }
  16.   },
  17.   "code": 30000, //其它非30000表示错误的情况
  18.   "message": "success",
  19.   "debug": false
  20. }

1.2 获取消息列表

需要登录：是
请求地址：http://api.artzhe.com/V30/ArtCircle/getMyMessageList
需要token：是(GET方式传token)

请求参数列表

字段 字段描述 规格 必须 示例 备注
page 页码 int 是 -
pagesize 每页获取条数 int 是 -

响应结果示例

  1. {
  2.   "data": {
  3.     "status": 1000 
  4.     "info":[
  5.        {
  6.         "id": "46696",//消息id

  10.         "type":1,//消息类型:14--用户收到发表艺术圈动态的评论，15--用户收到评论艺术圈动态的回复，16--用户收到发表艺术圈动态的喜欢
  11.         "content":"消息内容",//评论内容
  12.         "userInfo":{//评论或喜欢的人的信息
  13.             "uid":"12232",//用户id
  14.             "name":"老王",//消息发出者
  15.             "gender": "1", //用户性别1男2女3未知
  16.             "faceUrl": "http://artzhe.oss-cn-shenzhen.aliyuncs.com/test1.png", //头像
  17.             "isArtist": 1, //1表示是艺术家,0表示非艺术家
  18.             "isAgency": 1, //1表示是认证机构,0表示非认证机构
  19.             "isPlanner": 0, //1表示是策划者,0表示非策划者
  20.             "agencyType":7,//机构类型
  21.          },
  22.         "dynamic" : {//动态的信息
  23.             "type" : 1,//1纯文字2有图片3链接4视频

  25.             "content":"", //动态显示内容(type为1--文本前9个，type为2--图片链接，type为3--链接标题前9个，type为4--视频封面图缩略图链接)
